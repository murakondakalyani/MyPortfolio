MLFQ Visual Simulator

Operating Systems Scheduling Visualizer

An interactive educational simulator that visualizes Multi-Level Feedback Queue (MLFQ) CPU Scheduling through dynamic queue transitions, Gantt charts, performance metrics and process execution analysis.

Project Overview

The MLFQ Visual Simulator is an educational web-based application designed to help students understand Multi-Level Feedback Queue Scheduling in Operating Systems. The simulator visually demonstrates how processes move between queues, how CPU time is allocated and how different scheduling policies affect performance. It provides interactive process management, real-time scheduling visualization and performance analysis.

Problem Statement

CPU Scheduling is one of the most important concepts in Operating Systems. Students often find it difficult to understand how processes move between queues and how scheduling algorithms make decisions. The goal of this project was to create an interactive visual simulator that makes MLFQ Scheduling easier to learn and understand.

Understanding MLFQ

Multi-Level Feedback Queue Scheduling is an advanced CPU scheduling algorithm that uses multiple queues with different priorities. Processes start in the highest priority queue and may move to lower priority queues based on CPU usage. This approach balances responsiveness and fairness, making it suitable for modern operating systems.


Queue Structure

  • Queue 1 - Highest Priority
  • Queue 2 - Medium Priority
  • Queue 3 - Lowest Priority
  • Round Robin Scheduling
  • Priority-Based Execution

Simulation Workflow

  1. User enters process details.
  2. Processes are added to Queue 1.
  3. Scheduler allocates CPU time.
  4. Process execution begins.
  5. Queue transitions occur when needed.
  6. Execution is visualized.
  7. Gantt Chart is generated.
  8. Performance metrics are calculated.
  9. Results can be exported.
